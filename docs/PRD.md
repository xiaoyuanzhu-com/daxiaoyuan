# 大校园 PRD

## 名字

**大校园** —「大」取大同、大公、有容乃大之意，主张大学应向所有人敞开。

## 背景

新冠之后，国内大量高校对社会公众关闭。市民失去了在校园里散步遛弯、使用图书馆、操场、体育馆等公共资源的机会。围绕「大学是否应当向公众开放」的讨论持续多年，但缺少一个集中、可查、可比较的信息入口：

- 哪些大学完全开放？
- 哪些只开放散步、不开放设施？
- 哪些需要预约 / 凭身份证 / 凭校友卡？
- 周末和工作日是否不同？
- 暑期 / 寒假是否不同？

普通人遇到具体问题（"我想周末带孩子去 XX 大学操场跑步，能进吗？"）时，只能靠搜索碎片化的小红书帖子和过期新闻。

## 价值观

1. **开放** — 大学是公共资源，应当向社会敞开
2. **人人** — 不分校友、不分身份、人人可入
3. **透明** — 把现状摊开来给所有人看，是推动改变的第一步

## 形态

**首要：微信小程序**。轻量、即用、易传播，贴近真实使用场景（"在大学门口想知道能不能进"）。

**配套：H5 网页版**。便于在小程序之外的渠道分享（小红书 / 微博 / 浏览器直达），承担 SEO 入口与移动浏览器兼容。

小程序与 H5 共用同一套数据与大部分业务逻辑，通过跨端框架（如 Taro）一套代码编译两端。

## 核心功能（MVP）

### 1. 学校开放状态查询
- 列表 / 地图两种入口
- 每所学校一张卡片，展示：
  - 开放等级（完全开放 / 限时开放 / 预约开放 / 仅校友 / 关闭）
  - 设施开放情况（散步 / 图书馆 / 操场 / 体育馆 / 食堂）
  - 进入方式（刷身份证 / 预约 / 校友卡 / 自由通行）
  - 时间段（工作日 / 周末 / 寒暑假差异）
  - 最后更新时间

### 2. 用户共建
- 用户可补充 / 纠正信息（"我今天去过，实际是这样"）
- 简单的可信度机制：最近多人确认 > 旧信息

### 3. 倡导内容
- 一个固定页面解释「为什么大学应该开放」
- 已开放学校的故事、用户的真实使用画面

## 非目标（MVP 不做）

- 不做预约系统（学校自有渠道）
- 不做社交 / 评论 / 点赞
- 不做账号体系（共建用最轻方式：手机号 or 微信授权）
- 不覆盖中小学

## 信息架构

```
首页（地图 + 我附近的学校）
├── 学校详情页
│   ├── 开放状态
│   ├── 设施清单
│   ├── 进入方式
│   ├── 用户补充记录（时间倒序）
│   └── 「我去过，更新一下」按钮
├── 城市列表页
└── 关于大校园（价值主张）
```

## 数据来源与存储

**来源**
- **种子数据**：人工整理一线 + 新一线城市的 985/211 学校开放状态
- **持续更新**：用户共建为主
- **辅助来源**：抓取学校官网公告、小红书等公开内容做交叉验证

**存储**
- 学校静态信息以 JSON 文件存放在公开 GitHub 仓库，**按需加载**（首页只拉精简索引，进入详情页再拉对应学校的完整 JSON）
- 用户实时反馈走轻量后端（如微信云开发 / Cloudflare Workers）暂存，经审核后合入 GitHub
- 选择 GitHub 的原因：呼应"透明"价值观（数据公开、可审计）、版本化、欢迎社区 PR、免运维
- 国内访问通过 GitHub Action 同步到国内对象存储 / CDN，避免直连 raw.githubusercontent.com 的网络问题
